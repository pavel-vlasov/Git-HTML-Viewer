Core Components
📷

Test Report Builder

Single-file browser SPA. Ingests live captures, lets you edit payloads, validate JSON/XML, attach screenshots, and export dataset or report.

🔗

Report Viewer

GitHub-backed report browser. Sidebar tree with tag filtering by app/API/date/executor. Background sync every 5 min, manual force-refresh.

🔒

companion.js

Node.js MITM proxy. Intercepts Postman traffic, auto-generates a CA certificate, streams captured test cases via SSE to the builder in real time.

👁

Git HTML Viewer

Lightweight standalone viewer for any HTML file stored in a GitHub repo. No server required — renders reports directly from raw GitHub content. Share reports via link, no need to download files.

📦

Artifacts

dataset.json — DDT data for Postman re-runs. Execution bundle — comments & screenshots. HTML report — self-contained with trb-* meta tags.

🔨

Postman DDT + Newman Reporter

Data-driven Postman collection powered by dataset.json. Custom Newman Allure reporter captures per-request payloads and responses into structured log files.

Highlights
Live SSE Capture
Builder subscribes to /events stream from companion — new test cases appear in real time as Postman runs.
Git Integration
Push reports directly to GitHub from the builder. Configurable repo, branch, folder. Conflict-safe via SHA-based blob API writes.
Single-File SPA
Entire builder ships as one HTML file — zero build step, zero runtime dependencies. Open locally or serve statically.
Execution Comments & Images
Attach notes and base64 screenshots per test case. Stored in a separate execution bundle, excluded from dataset export.
Smart API Path Normalization
UUID/numeric path segments replaced with {id} in tags and viewer filters for clean deduplication across runs.
Jira Linkage
Attach Jira ticket URLs to test cases. Rendered as clickable badges in the report and surfaced as filter tags in the viewer.
